# JS_TS_reading_order.md

## Reading order for your current stack

This is the cleanest order to build from fundamentals to architecture without too much overlap.

### Phase 1 — language foundation
1. **Eloquent JavaScript**  
   Rebuild JavaScript runtime intuition: scope, closures, objects, async basics, and general fluency.

2. **Programming TypeScript**  
   Layer TypeScript properly on top of JavaScript: types, interfaces, generics, modules, and practical TS structure.

3. **Effective TypeScript**  
   Tighten your TypeScript thinking. This is where you improve judgment, not just syntax.

---

### Phase 2 — design and application structure
4. **Learning JavaScript Design Patterns**  
   Move from syntax into code organization, reuse, composition, and application structure.

5. **Event Based JavaScript Design**  
   Strengthen event-driven thinking, which matters a lot in JavaScript, Node.js, and async systems.

6. **Node.js Design Patterns**  
   One of the highest-value books in the stack for real-world backend and asynchronous architecture.

---

### Phase 3 — frontend
7. **Learning React**  
   Once JS + TS are stable, move into React. This will land much better after the earlier books.

---

### Phase 4 — depth and reference
8. **JavaScript: The Definitive Guide**  
   Use this partly as a read and partly as a long-term reference. Do not force it cover to cover too early.

9. **JavaScript: The Good Parts**  
   Read this as a short perspective book. Useful for taste and discipline, but treat it as partly historical.

---

### Phase 5 — optional / situational
10. **Data Structures and Algorithms with JavaScript**  
    Read if interview prep becomes important.

11. **Pro JavaScript Design Patterns**  
    Read only if you want more depth after the earlier design-pattern book.

12. **Building Microservices**  
    Read once you want to think beyond application code and more about service boundaries and system design.

---

## Short version

If you want the highest-value sequence only:

1. **Eloquent JavaScript**
2. **Programming TypeScript**
3. **Effective TypeScript**
4. **Learning JavaScript Design Patterns**
5. **Node.js Design Patterns**
6. **Learning React**
7. **Building Microservices**

---

## Practical reading advice

- Do not read all of them straight through without coding.
- After each book, build something small.
- Use **JavaScript: The Definitive Guide** mainly as a reference companion.
- Keep **Effective TypeScript** nearby even after you finish it.
